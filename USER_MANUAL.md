> 本手冊整理所有可用指令與使用情境，供業務、助理、管理員查閱。
> 最後更新：2026-04-22

---

## ⚠️ 基本規則

1. **群組中必須標註 `@KINYO挺好的`**，機器人才會回應（1 對 1 私訊可省略）
2. **指令大小寫不敏感**，但型號請盡量用正規寫法（例：`KPB-2990`）
3. 所有通知信、LINE 推播會送至「接單總部」群組與助理信箱（din@kinyo.tw）

回傳下單模板，填寫完整後送出：

```

送貨地址：
```

送出後會出現 **Flex 確認卡**，按「✅ 確認無誤」即寄出 email 通知並推播接單總部。

### 📌 預留訂單（佔位先留貨）

**指令**：`預留訂單` 或 `留貨`

**用途**：客戶已確認要買但還不到出貨時間，先把貨留起來避免被別的單消耗。

```
@KINYO挺好的 預留訂單

採購公司：
收件人：
聯絡電話：
送貨地址：
預留期限：（例：2026/05/31）
備註：
=================
型號： / 數量： / 單價：  (單價沒有可空白)
型號： / 數量： / 單價：
```

**系統行為**：
- 按「✅ 確認預留」後建立 `RSV` 開頭的預留單號
- 寄信給助理（sam、din、chloe、kelly）
- 推播至接單總部
- **預留期限前 7 天早上 9 點**系統自動再寄一次提醒信，要助理準備做單出貨

### 🔍 商品查價

**直接輸入關鍵字**（群組記得加 `@KINYO挺好的`）：

- `KPB-2990` — 查單一型號
- `吹風機 預算500-700 數量300` — 自訂條件查價
- `KVC-6243 數量100 庫存大於50` — 指定庫存條件

**模糊預算處理**：
- `1000 左右` → 自動抓 ±10% → 900~1100
- `預算1000` → 視為上限 → 0~1000

## 🔧 常見問題 FAQ

### Q1：機器人為什麼沒反應？
- 群組裡**沒有標註 `@KINYO挺好的`** → 機器人會靜默忽略
- 該群組**未綁定**或**查價已關閉** → 請管理員執行 `#綁定群組` + `#開放查價`

### Q2：助理說收不到通知信？
1. 檢查垃圾信件夾
2. 在信箱搜尋 `from:sam.kuo@kinyo.tw`
3. 若完全搜不到，請 IT 把 `sam.kuo@kinyo.tw` 加進白名單

### Q3：預留訂單過期了怎麼辦？
系統會在**期限前 7 天**自動寄信提醒。收到後請 14 天內處理：
- 轉成正式訂單 → 用 `訂單` 模板重開一張
- 釋放庫存 → 直接在 Firestore `ReservedOrders` 集合把 status 改成 `cancelled`

### Q4：查價出現「無法辨識指令」？
請改用標準格式：
- 查價：`查價 + 型號`
- 下單：先打 `訂單` 取模板
- 維修：直接打 `維修` 或 `客服`

### Q5：如何換接單總部群組？
1. 把 `@KINYO挺好的` bot 加到新群組
2. 管理員（sam）在新群組輸入 `@KINYO挺好的 設定為接單總部`
3. 舊群組自動失效，新群組開始接推播
